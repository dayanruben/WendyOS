## Security Overview

This page describes the current security posture of Wendy and WendyOS and what's coming next.

<Callout type="warn">
  **Access control and device enrollment are not yet available.** Treat any reachable agent as accessible to any authenticated CLI on the same network or tunnel. Do not expose wendy-agent to untrusted networks until access control ships.
</Callout>

## Current Status

| Area | Status |
|------|--------|
| CLI-to-Agent communication | Authenticated |
| Data in transit | mTLS encrypted with Post-Quantum Cryptography |
| Cloud connectivity | Zero-trust — PKI authentication required |
| Device identity / Enrollment | Nearing launch |
| Access control | Nearing launch |

### Safe Usage Guidelines

<Cards>
  <Card icon={<ShieldCheck />} title="Recommended">
    Private networks, home labs, classrooms, hackathons, development desks, and any environment where all CLI users are trusted.
  </Card>
  <Card icon={<ShieldAlert />} title="Not Recommended">
    Public or shared networks, production deployments, sensitive data or workloads, and internet-facing devices — until access control ships.
  </Card>
</Cards>

**General recommendations:**

- Run Wendy devices on a private or isolated network
- Do not expose wendy-agent directly to the internet
- Do not deploy applications that handle credentials, personal data, or sensitive information before access control is available

## Security Roadmap

### Secure Device Identity and Enrollment

Devices go through a secure onboarding process with cryptographic identity. Once launched, this enables trusted enrollment into fleets and prevents unauthorized devices from joining your infrastructure.

### Project-Scoped Access Control

Granular control over who can deploy what and where. CLI instances and users will be scoped to specific projects and devices, so you can share a network without sharing access.

## Reporting Security Issues

If you discover a security vulnerability, please report it responsibly by emailing **security@wendylabs.com** rather than filing a public issue. We take all reports seriously and will respond promptly.
