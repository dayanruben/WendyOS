Streams all OTel data as a custom JSONL format. This is used by the [VSCode extension](../../../../vscode/) to render the OTel traces, metrics and logs using only a single API call.

## Planned: runtime telemetry for embedded apps (not yet shipped)

> The flow described below is **planned, not implemented**. Today the CLI only streams the OTel data generated by `wendy-agent` itself; runtime failures inside an embedded WASM app are not yet surfaced back to the CLI automatically.

A pending change will pipe runtime logs and errors from on-device apps — including heap-allocation failures, WAMR fault conditions, and other runtime issues that pre-deploy validation can't catch — back to the user through this same CLI surface. Memory limits and the exact taxonomy of reported errors are still being defined.
