## How integrations work

Both integrations build on the same foundation. When you deploy a ROS 2 app —
with a `frameworks.ros2` block in `wendy.json`, or a `network_mode: host` service
in a [multi-container deployment](/docs/guides/multi-app-deployments) — the agent
discovers the app's containers and starts a lightweight CLI sidecar in the same
DDS domain. Inspection and bridging commands run inside that sidecar over the
CLI's existing mTLS gRPC connection.

That means you get full `ros2` tooling and live visualization without opening a
shell on the device, sourcing environment files, or exposing any extra ports.

## Get started

1. Deploy a ROS 2 app. See [Multi-app deployments](/docs/guides/multi-app-deployments)
   for a complete talker/listener stack and the host-networking entitlement ROS 2
   discovery needs.
2. Inspect it live with [Wendy for ROS 2](/docs/integrations/ros2).
3. Visualize it in [Foxglove](/docs/integrations/foxglove) *(coming soon)*.
